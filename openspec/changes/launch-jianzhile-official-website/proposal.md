## Why

当前仓库只有 OpenSpec / spec-workflow 脚手架，没有任何官网应用代码、部署配置、SEO 资产、埋点体系或表单承接能力，但 PRD 已明确要求尽快上线「简职了」官方网站，以支撑品牌建立、B 端试用/演示转化、C 端注册转化与信任背书。现在启动该变更，可以把“仅有产品需求”的状态推进到“具备可规划、可实现、可验证的官网能力边界”，避免在工程初始化、转化闭环、合规与 SEO 上后期返工。

## What Changes

- 新增简职了官网的 P0 需求范围定义：首页、B 端解决方案页、定价页、申请试用/联系我们页、关于我们页。
- 新增官网基础能力约束：从零建立站点工程基线、页面路由结构、共享导航/页脚、响应式体验与内容承载能力。
- 新增 B 端核心转化闭环要求：站内提交「申请试用 / 预约演示」表单，形成可验证的提交成功/失败/线索承接路径。
- 新增 C 端转化入口要求：官网保留双端分流，并支持 C 端官网内提交；但由于首发范围限定为 P0，C 端在 P0 中采用独立页面、轻量注册页还是首页内嵌提交，仍需在 planning 阶段最终收敛。
- 新增站点级信任与增长要求：基础 SEO、结构化数据、隐私/协议入口、公司主体信息、GA4 + Hotjar 埋点与关键转化事件观测。
- 明确当前不包含 P1/P2 范围：客户案例页、博客/资源中心、完整版 C 端产品介绍页不纳入本次首发实现。

## Capabilities

### New Capabilities
- `official-website-foundation`: 官网工程基线、共享布局、导航页脚、P0 路由与响应式承载能力。
- `home-and-audience-routing`: 首页品牌表达、双端价值传达与 B/C 端 CTA 分流能力。
- `employer-conversion-funnel`: B 端解决方案、定价、联系我们/试用页，以及站内表单提交闭环。
- `candidate-conversion-entry`: C 端官网内转化入口与提交承接能力，具体 P0 交互形态待 planning 阶段收敛。
- `trust-seo-and-insights`: SEO 基线、结构化数据、法律与主体信息入口、GA4 + Hotjar 事件观测能力。

### Modified Capabilities
- 无

## Impact

- 受影响系统：官网前端应用、站内表单提交接口/数据承接层、分析埋点体系、SEO 资产、法律/合规内容、部署配置。
- 受影响依赖：Next.js 15、TailwindCSS、shadcn/ui、Framer Motion、react-hook-form、zod、GA4、Hotjar、Vercel（均为当前规划约束，仓库内尚未落地）。
- 受影响流程：B 端试用/演示线索收集、C 端注册线索收集、页面 SEO 收录、转化漏斗观测。
- 关键前置风险：当前仓库无应用基线、无 Git/CI、无现成部署配置、无法律文案与分析事件规范、无已定义的表单数据模型与持久化方案。
